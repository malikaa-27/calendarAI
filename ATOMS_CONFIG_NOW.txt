=== getAvailableSlots - LLM PARAMETER (REQUIRED) ===
Add this so the day/time the caller says gets passed to the backend:

  Parameter name: day_time_mentioned_by_user
  Description: "The day and time the caller wants. Extract exactly what they said. Examples: today 2 pm, tomorrow, Friday 3 pm, next Monday, March 15, March 15 2026, March 15th next year, 12/25/2026, next week Tuesday. Any day - this week, next year, or specific date."

Without this, we get fallback slots (wrong day).

=== RESPONSE VARIABLE EXTRACTION (getAvailableSlots) ===
  Variable: available_summary    Path: $.available_summary
  Variable: selected_slot_start_iso   Path: $.first_slot_start
  Variable: selected_slot_end_iso     Path: $.first_slot_end

  CRITICAL - REMOVE these if you have them (they cause "NoneType" errors):
  - $.available[0].start  → use $.first_slot_start instead
  - $.available[0].end    → use $.first_slot_end instead
  - $.formatted, $.slots  → remove entirely
  Variable name must be exactly "selected_slot_end_iso" with NO trailing space.
